Submitting Issues - please complete the form using the template on the Issues page of Github. Fill in as much information as possible, but not every field is mandatory.
 
Proposing features - discuss with team members first. If ageed upon, create a card on Zube.io

Branches - new features must be done in a new branch titled 'feature/[name]'. Once complete, merge into Develop. 
  Development in Develop is not recomended, branches Release and Master must NOT be developed in.

Testing - please carry out unit testing alongside your work. Thurthermore, integration testing must be carried out,
  this is not required for every change but when your code affects multiple components.
